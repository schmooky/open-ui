The whole HUD is one **`UISpec`** ‚Äî a plain, JSON-serializable object you pass to
`mountHud(app, spec)`. Every field is optional; omit one and you get the sensible
default. It's fully typed, so your editor autocompletes the options and a typo
won't compile.

```ts
import { mountHud } from '@open-ui/pixi';

const hud = mountHud(app, {
  theme: 'midnight',
  turbo:    { modes: 3 },
  autoplay: { mode: 'options' },
  spin:     { press: 'hold-to-spin' },
});
```

`mountHud()` with no spec gives you the reference HUD. Everything below is opt-in.

## Theme

```ts
theme: 'default' | 'midnight' | 'neon'
// or, to tweak a preset (overrides are a SAFE, curated subset):
theme: { preset: 'neon', overrides: { color: { accent: '#ff4d6d' }, radius: { card: 20 } } }
```

Three presets ship in the box. Overrides are limited to colours, corner radii and
the font family, and every value is validated ‚Äî a bad colour falls back to the
token, so a theme can restyle the HUD but **cannot break it**. See
[Theming](/guides/theming/).

## Turbo ‚Äî 2-mode or 3-mode

The single Turbo control is a 2-mode toggle **or** a 3-mode switcher, by config.
Each press advances to the next mode and wraps back to off.

| `turbo.modes` | Result | Cycle |
| --- | --- | --- |
| `2` *(default)* | off / on toggle | off ‚Üí on ‚Üí off |
| `3` | three-mode switcher | off ‚Üí turbo ‚Üí super ‚Üí off |
| `['off','fast','ludicrous']` | your own ladder (first = off) | off ‚Üí fast ‚Üí ludicrous ‚Üí off |

```ts
turbo: { modes: 3, index: 0 }   // start at "off"
```

The renderer draws a small level indicator (lit pips) for 3+ modes, so "turbo" vs
"super" reads at a glance even on 2-frame art. Each change emits a `turboChanged`
event (plus a back-compatible `toggled`).

## Autoplay ‚Äî options drawer or infinite

```ts
autoplay: {
  mode: 'options' | 'infinite',   // default 'options'
  options: [5, 10, 25, 50, 100, Infinity],
}
```

- **`'options'`** ‚Äî tapping autoplay slides a **drawer up from the bottom** with
  the count choices and a **Start** button. Pick a count, press Start.
- **`'infinite'`** ‚Äî tapping starts infinite autoplay immediately; tap again to
  stop. No picker.

`Infinity` in `options` renders as **‚àû**. The library owns the picker and the live
remaining count; **your game runs the loop** and feeds it via `hud.ui.autoplay.setCount(n)`.
See the [autoplay recipe](/recipes/autoplay-loop/).

## Spin ‚Äî tap or hold-to-spin

```ts
spin: { press: 'tap' | 'hold-to-spin' }   // default 'tap'
```

With **`'hold-to-spin'`**, holding the spin button fires `holdSpinStarted`; your
game turbo-spins on a loop until release, which fires `holdSpinStopped`. A quick
tap still spins once. (The library signals intent; the host runs the loop ‚Äî same
pattern as autoplay.)

```ts
hud.on('holdSpinStarted', () => turboLoop.start());
hud.on('holdSpinStopped', () => turboLoop.stop());
```

## Responsive ‚Äî per device & orientation

Layer overrides on top of the base layout, keyed by **device size** (`mobile`,
`tablet`, `desktop`) or **orientation** (`portrait`, `landscape`). They re-apply on
every resize, so the HUD can reflow or drop controls per screen.

```ts
responsive: {
  portrait: {
    controls: {
      spin:    { layout: { anchor: 'bottom-center', offset: [0, -470] } },
      autoplay:{ layout: { anchor: 'bottom-center', offset: [-150, -250] } },
    },
  },
  mobile: { controls: { bonus: { hidden: true } } },  // drop the bonus button on phones
}
```

Order of precedence is **base ‚Üí orientation ‚Üí size** (size wins). Device size is
read from the **shorter screen edge** (`‚â§ 480` mobile, `‚â§ 840` tablet, else desktop ‚Äî
configurable via `layout.breakpoints`), so a phone stays "mobile" in both
orientations. See [Responsive layout](/guides/layout/).

## Money ‚Äî currency & bet ladder

```ts
currency: { code: 'BTC', decimals: 8 },          // sets balance + bet at once
betLadder: { levels: [0.5, 1, 2, 5, 10, 20], index: 1 },
```

`decimals` up to 8 (crypto-friendly); bad host amounts are clamped, never
rejected, so a stray value can't corrupt the display.

## Per-control overrides

Target any control by id to move it, hide it, disable it, or relabel it:

```ts
controls: {
  bonus: { hidden: true },
  turbo: { disabled: true },
  spin:  { layout: { anchor: 'bottom-center', offset: [0, -440] } },
}
```

Hidden controls stay **registered** ‚Äî still in `snapshot()` for tests ‚Äî they're
just not drawn.

## The menu ‚Äî one scrollable sheet

The ‚ò∞ button opens a single scrollable menu with three sections, in order:
**Settings ‚Üí Paytable ‚Üí Rules**. Music + Sound (and a Language switch, when 2+
locales are set) ship out of the box; you add your own.

```ts
menu: {
  banner: { src: '/art/title.png', width: 560, height: 96 },        // themed header art
  settings: [{ kind: 'toggle', id: 'quick', label: 'Quick spin' }], // appended to Music/Sound
  paytable: [{
    kind: 'paytable', id: 'pt', columns: 2,                         // a multi-column grid
    rows: [{ symbol: 'Wild', icon: '/art/wild.png', payouts: '8-9: 10x\n10-11: 25x\n12+: 50x' }],
  }],
  rules: [
    { kind: 'text', id: 'r', text: 'Match symbols to win ‚Äî **bigger symbols pay more**.' }, // **bold** inline
    { kind: 'stat-grid', id: 's', items: [{ label: 'RTP', value: '96.50%' }, { label: 'Max win', value: '5,000x' }] },
    { kind: 'callout', id: 't', tone: 'bonus', title: 'Tip', text: 'Hold spin for turbo.' },
  ],
},
```

**Everything is a declarative block** ‚Äî a discriminated union (`slider`, `toggle`,
`select`, `stepper`, `heading`, `text`, `callout`, `stat-grid`, `steps`,
`paytable`, `image`), so "shown but no payload" can't compile. Headings render as a
centered divider; `text`/paytable payouts support `**bold**`; paytable takes a
`columns` grid with per-symbol `icon`s; every block's text flows through `ui.t`, so
the whole menu localizes.

## i18n ‚Äî localize safely

Pass a dictionary and starting locale; a Language switch appears in the menu
automatically.

```ts
locale: { messages: { de: { Balance: 'Guthaben', Bet: 'Einsatz' } }, locale: 'en' },
```

`hud.ui.setLocale('de')` re-renders every label. **Keys fall through**, so a partial
dictionary can never produce blank or broken UI ‚Äî it just shows the key. (The
example client ships a 10-locale dictionary, screenshotted by a Playwright matrix.)

## It never throws

The spec is validated on mount. Problems are **reported, not fatal** ‚Äî a malformed
HUD still boots with the bad pieces dropped. Catch issues in development:

```ts
mountHud(app, spec, { hooks: { onDataIssue: (i) => console.warn(i.code, i.message) } });
```
